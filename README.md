# IndyCar Race Position Predictor

An interactive machine learning app that predicts IndyCar finishing positions for the 2026 season. Built with PyCaret and Streamlit, the app supports both pre-qualifying and post-qualifying predictions for full race simulations.

🔗 **[Live Demo](https://rubens-indycar-predictor.streamlit.app/)** 

---

## How It Works

The app uses two separate regression models trained on historical IndyCar data (2012–2026):

- **Pre-Qualifying Model** — predicts finishing positions before qualifying, using only historical driver, team, and track performance metrics
- **Post-Qualifying Model** — predicts finishing positions after qualifying, incorporating grid position as an additional feature

Both models output a normalized position score (0–1) which is denormalized to a final finishing position (1–25).

### Feature Hierarchy

For each driver the app builds a feature row using a three-level lookup hierarchy:

| Level | Known Track | New Track |
|-------|------------|-----------|
| Most specific | Track-specific stats `(DriverID, TrackID)` | Track-type stats `(DriverID, TrackTypeID)` |
| Fallback | Track-type stats | Overall career stats |
| Last resort | Overall career stats | Dataset averages |

Rolling metrics like `DNFRate`, `DriverElo`, and `DRFAvg` always pull from the driver's most recent race regardless of track, while track-specific metrics like `DTAvg` and `DriverTElo` follow the full hierarchy.

Rookies use their team's historical data as a proxy, following the same hierarchy with `(TeamID, TrackID)` → `(TeamID, TrackTypeID)` → team overall → rookie averages.

## Feature Engineering

All features are generated by `metrics_generator.ipynb` which processes raw race results into rolling and Elo-based metrics. The notebook must be re-run after each race to update the dataset before retraining.

### Driver Features

| Feature | Description | Calculation |
|---------|-------------|-------------|
| `DRFAvg` | Driver Rolling finishing average | Last 5 races, normalized position |
| `DTAvg` | Driver Track finishing average | Expanding mean at `(DriverID, TrackID)` |
| `DTTAvg` | Driver Track type finishing average | Expanding mean at `(DriverID, TrackTypeID)` |
| `DNFRate` | DNF rate | Rolling 20-race window |
| `TDNFRate` | Track-specific DNF rate | Expanding mean at `(DriverID, TrackID)` |
| `DriverElo` | Overall Elo rating | K=64, updated each race |
| `DriverTElo` | Track-specific Elo | K=64, updated at `(DriverID, TrackID)` |
| `DriverTTElo` | Track-type Elo | K=64, updated at `(DriverID, TrackTypeID)` |
| `DriverRitmo` | Form/momentum metric | Exponentially weighted 5-race mean |

### Team Features

| Feature | Description | Calculation |
|---------|-------------|-------------|
| `TRP` | Team rolling performance | Last 5 races mean normalized finish |
| `TTP` | Team track performance | Expanding mean at `(TeamID, TrackID)` |
| `TeamDNFRate` | Team DNF rate | Rolling 20-race window |
| `TeamElo` | Overall team Elo | K=48, updated each race |
| `TeamTElo` | Track-specific team Elo | K=48, updated at `(TeamID, TrackID)` |
| `TeamRitmo` | Team form/momentum | Exponentially weighted 5-race mean |

### Engine Features

| Feature | Description | Calculation |
|---------|-------------|-------------|
| `EngineElo` | Overall engine Elo | K=32, updated each race |
| `EngineTElo` | Track-specific engine Elo | K=32, updated at `(EngineID, TrackID)` |
| `EngineTTElo` | Track-type engine Elo | K=32, updated at `(EngineID, TrackTypeID)` |

### Elo System

All Elo ratings use a pairwise comparison system adapted for multi-competitor racing. Each driver/team/engine is compared against every other competitor in the same event, with scores normalized by field position. Different K-factors reflect confidence in each entity's consistency — drivers (64) are most reactive, engines (32) most stable.

## Models

| Phase | Algorithm | Unseen MAE | Unseen R² |
|-------|-----------|------------|-----------|
| Pre-Qualifying | Random Forest + CatBoost Blend | 0.2222 | 0.1939 |
| Post-Qualifying | CatBoost + LightGBM + Bayesian Ridge Blend | 0.2081 | 0.2489 |

Models were trained using PyCaret with TimeSeriesSplit (10 folds) to respect temporal ordering of race data.

## Workflow

After each race:
1. Add raw results to the base dataset (PositionFinish, PositionStart, Status etc.)
2. Run `metrics_generator.ipynb` to recompute all rolling and Elo features
3. Retrain models and compare MAE/R² on unseen data
4. Deploy new model only if it improves on previous performance
